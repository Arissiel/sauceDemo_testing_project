# Функциональное и исследовательское тестирование SauceDemo

## Цель тестирования

Провести функциональное и исследовательское тестирование демо-приложения SauceDemo, проверить основные пользовательские сценарии, выявить возможные дефекты и оценить поведение системы при использовании различных тестовых пользователей.

---

# Область тестирования

В ходе тестирования были исследованы следующие области приложения:

* авторизация пользователей;
* обработка ошибок авторизации;
* управление пользовательской сессией;
* доступ к защищённым разделам приложения;
* каталог товаров;
* корзина покупок;
* производительность интерфейса;
* визуальное отображение элементов интерфейса.

---

# Использованные тестовые пользователи

| Пользователь            | Цель проверки                  |
| ----------------------- | ------------------------------ |
| standard_user           | Эталонное поведение системы    |
| locked_out_user         | Проверка ограничений доступа   |
| visual_user             | Поиск визуальных дефектов      |
| error_user              | Проверка функциональных ошибок |
| performance_glitch_user | Анализ производительности      |

---

# Анализ рисков

## Риск 1. Пользователь не может получить доступ к приложению

### Проверки

* вход под standard_user;
* повторный вход после logout;
* вход из другого браузера.

### Результат

Проблем не обнаружено.

---

## Риск 2. Возможен несанкционированный доступ

### Проверки

* неверный логин;
* неверный пароль;
* неверный логин и пароль;
* пустые поля;
* locked_out_user;
* прямой переход на защищённые URL;
* попытка обхода клиентских ограничений.

### Результат

В рамках доступной функциональности демо-приложения проблем не найдено.

---

## Риск 3. Некорректная работа пользовательской сессии

### Проверки

* обновление страницы после входа;
* logout + возврат кнопкой Back;
* открытие новой вкладки;
* закрытие браузера.

### Результат

Поведение соответствует ожидаемому.

---

## Риск 4. Снижение производительности пользовательских сценариев

### Проверки

Сравнение поведения standard_user и performance_glitch_user при выполнении пользовательских действий.

### Результат

Обнаружены существенные задержки при выполнении операций и навигации по приложению.

Оформлен отдельный баг-репорт.

---

## Риск 5. Некорректное отображение интерфейса

### Проверки

Использование visual_user для сравнения интерфейса с эталонным пользователем.

### Результат

Обнаружены визуальные дефекты отображения товаров и элементов интерфейса.

Оформлен отдельный баг-репорт.

---

## Риск 6. Нарушение работы бизнес-функциональности

### Проверки

Использование error_user для выполнения операций с товарами и корзиной.

### Результат

Обнаружены ошибки при работе с корзиной.

Оформлен отдельный баг-репорт.

---

# Исследовательское тестирование

В процессе тестирования дополнительно проверялись гипотезы о поведении системы.

## Проверенные гипотезы

### После попытки входа заблокированного пользователя создаётся пользовательская сессия

Результат: не подтвердилось.

---

### Возможен доступ к защищённым страницам без авторизации

Результат: не подтвердилось.

---

### Различные типы пользователей могут демонстрировать различное поведение системы

Результат: подтвердилось.

Обнаружены различия в производительности, отображении интерфейса и работе корзины.

# Ограничения тестирования

В рамках доступного функционала не удалось определить:

* максимальную длину логина;
* максимальную длину пароля;
* срок жизни пользовательской сессии;
* наличие механизма блокировки после серии неудачных попыток входа.

---

# Обнаруженные дефекты

## Bug #1

Существенное снижение производительности для пользователя performance_glitch_user.

Severity: Major

---

## Bug #2

Некорректное отображение изображений товаров и элементов интерфейса для пользователя visual_user.

Severity: Major

---

## Bug #3

Некорректная работа корзины для пользователя error_user.

Severity: Major

---

# Итоги тестирования

Проверены основные пользовательские сценарии приложения, включая авторизацию, работу с каталогом товаров, корзиной и пользовательской сессией.

В ходе тестирования выявлены:

* 3 дефекта уровня Major;
* визуальные дефекты интерфейса;
* функциональные ошибки работы корзины;
* существенные проблемы производительности отдельных сценариев.

Критических проблем, связанных с доступом к системе и управлением сессией, не обнаружено.

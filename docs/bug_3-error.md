# Bug #3. Некорректная работа функций Add to cart и Remove для пользователя error_user

## Информация о дефекте

**Severity:** Major

**Priority:** High

**Status:** New

**Окружение:**

* Google Chrome Version 149.0.7827.156 (64-bit)
* Windows 11

---

## Описание

После авторизации под пользователем `error_user` часть товаров невозможно корректно добавлять в корзину и удалять из корзины со страницы каталога товаров и со страницы отдельного товара.

Проблема воспроизводится стабильно для затронутых товаров.

---

## Предусловия

Пользователь находится на странице авторизации SauceDemo:

https://www.saucedemo.com/

---

## Шаги воспроизведения

1. Открыть страницу авторизации SauceDemo.
2. Ввести:

   * Username: `error_user`
   * Password: `secret_sauce`
3. Нажать кнопку **Login**.
4. На странице товаров попытаться добавить различные товары в корзину с помощью кнопки **Add to cart**.
5. Для успешно добавленных товаров попытаться удалить их с помощью кнопки **Remove**.
6. Перейти на страницу отдельного товара и повторить операции добавления и удаления.

---

## Ожидаемый результат

Для любого товара:

* кнопка **Add to cart** добавляет товар в корзину;
* после добавления отображается кнопка **Remove**;
* кнопка **Remove** удаляет товар из корзины;
* функциональность одинаково работает на странице каталога и на странице отдельного товара.

---

## Фактический результат

Для части товаров:

* кнопка **Add to cart** не добавляет товар в корзину;
* кнопка **Remove** не удаляет ранее добавленный товар со страницы каталога;
* кнопка **Remove** также не работает на странице отдельного товара.

При этом удаление товара из корзины работает корректно на странице Cart.

---

## Частота воспроизведения

100% для затронутых товаров.

---

## Влияние на пользователя

Пользователь не может полноценно управлять составом корзины из основных пользовательских сценариев приложения.

Функциональность покупки товаров нарушена частично.
